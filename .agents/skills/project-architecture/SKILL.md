# Project Architecture

This skill provides a high-level map of the Next.js template ecosystem, built on Next.js 16.1+ App Router and Tailwind CSS v4.

## High-Level Folder Structure

**Important for AI Agents:** The exact subdirectories may vary. **Always list the root directory and `src/` to understand the specific structure of the current fork.**

Common architecture examples include:

```text
├── .agents/skills/    # AI Agent skills (like this document)
├── public/            # Static assets (images, fonts, favicon)
├── scripts/           # Node scripts (theme generation, LLM docs generation)
├── src/
│   ├── app/           # Next.js App Router pages and layouts
│   ├── config/        # Global JSON configuration files
│   ├── content/       # Markdown/MDX content database
│   ├── hooks/         # Custom React hooks
│   ├── layouts/       # Reusable UI Components and Partials
│   ├── lib/           # Utility functions (markdown parsers, formatters)
│   ├── styles/        # Tailwind v4 CSS configuration and custom styles
│   └── types/         # TypeScript definitions
```

## Data Flow: From Content to Page

This template functions as a Git-based CMS. Here is how data moves through the system:

1. **Content Definition**: A user creates `src/content/blog/my-post.md`.
2. **Data Parsing**: Functions inside `src/lib/` (e.g., `getSinglePage`, `getRegularPage`) are called by Next.js Server Components. These functions read the file system, parse the frontmatter using `gray-matter`, and return structured JSON.
3. **Routing**: The `src/app/blog/[single]/page.tsx` dynamic route catches the URL `/blog/my-post`.
4. **Rendering**: The page component receives the parsed markdown data, renders the HTML using `next-mdx-remote` or `marked`, and wraps it in UI components from `src/layouts/`.

## What to Edit vs. What to NEVER Touch

### ✅ Safe to Edit
- **`src/content/**`**: Where all text and image content lives.
- **`src/config/**`**: Where global settings, menus, and theming (colors/fonts) are defined.
- **`src/app/**`**: Add new routes or modify page templates here.
- **`src/layouts/**`**: Modify or add UI components.

### 🚫 NEVER Touch (Without Extreme Caution)
- **`src/styles/generated-theme.css`**: Auto-generated by scripts. Your edits will be overwritten instantly.
- **`src/lib/` markdown parsers**: Unless you are deliberately changing the core mechanics of how `.md` files are read, leave these alone. They are highly dependent on the exact folder structure in `src/content/`.
- **`scripts/`**: The build and dev scripts. Modifying `themeGenerator.mjs` or `jsonGenerator.mjs` can break the entire CSS pipeline.

## Build Pipeline Architecture

When `pnpm dev` or `pnpm build` is executed, the `package.json` scripts trigger `concurrently` or sequential execution of:
1. `node scripts/themeGenerator.mjs` -> Compiles `theme.json` to `generated-theme.css`.
2. `node scripts/jsonGenerator.mjs` -> Often generates a search index (`search.json`) from markdown files.
3. `next dev` / `next build` -> The standard Next.js process.
4. `node scripts/llmsGenerator.mjs` -> (Post-build) generates `llms.txt` for AI agent consumption based on `config.json` rules.
