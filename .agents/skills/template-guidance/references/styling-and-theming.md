# Styling and Theming

This template uses **Tailwind CSS v4** combined with a custom **dynamic theming engine** powered by JSON configurations.

## Theming Engine (`theme.json`)

The color palette and typography are controlled centrally via `src/config/theme.json`.

**Important for AI Agents:** The exact schema of `theme.json` may have been customized. **Always read `src/config/theme.json` to see the current color and font structure before making changes.**

### How it works
There is a Node.js script (`scripts/themeGenerator.mjs`) that reads `theme.json` and auto-generates `src/styles/generated-theme.css`. This script runs automatically during `npm run dev` and `npm run build`.

### Changing Colors
Edit `src/config/theme.json`:
```json
"colors": {
  "default": {
    "theme_color": {
      "primary": "#121212",
      "body": "#fff",
      "border": "#eaeaea"
    }
  },
  "darkmode": { ... }
}
```
*These values are converted into CSS variables like `--color-primary` and `--color-darkmode-primary`.*

### Changing Fonts
Edit `src/config/theme.json`:
```json
"fonts": {
  "font_family": {
    "primary": "Heebo:wght@400;600",
    "primary_type": "sans-serif"
  }
}
```

## Tailwind CSS v4 Integration

The main Tailwind configuration is now entirely CSS-driven in Tailwind v4. The entry point is `src/styles/main.css`.

### `src/styles/` Directory Structure
- `main.css`: The root file importing Tailwind and all custom layers.
- `generated-theme.css`: Auto-generated by `themeGenerator.mjs`. **DO NOT EDIT.**
- `base.css`: Base HTML element styling.
- `components.css`: Custom component classes (if Tailwind utilities are not enough).
- `utilities.css`: Custom utility classes.
- `navigation.css`, `buttons.css`, `search.css`: Specific component styles.

## Dark Mode

Dark mode is handled using `next-themes` and a custom Tailwind variant.
- The `src/styles/main.css` defines `@custom-variant dark (&:where(.dark, .dark *));`.
- When in dark mode, the `next-themes` provider adds a `.dark` class to the HTML element.
- The CSS variables generated for `darkmode` in `theme.json` automatically apply.

To use dark mode specific Tailwind classes, prefix them with `dark:` (e.g., `dark:bg-darkmode-body`).

## Common Mistakes / What NOT to do

- **DO NOT EDIT `src/styles/generated-theme.css` MANUALLY.** Any changes will be immediately overwritten by `themeGenerator.mjs` the next time the app builds or runs in dev mode.
- **DO NOT** hardcode hex colors in your React components or Tailwind classes (e.g., `text-[#121212]`). Always use the theme variables provided (e.g., `text-primary`).
- **DO NOT** configure Tailwind in a `tailwind.config.js` file. Tailwind v4 uses CSS for configuration. If you need to add custom utilities, add them to `src/styles/utilities.css` or use the `@theme` directive in `main.css`.
